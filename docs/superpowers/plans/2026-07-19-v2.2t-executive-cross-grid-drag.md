# v2.2T Executive Cross-grid Drag Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let authorized users drag an Executive milestone across Sections and Quarters, while labels and access policies are configured by stable IDs rather than fixed displayed names.

**Architecture:** Add a versioned `executiveMilestoneConfig/timeline` document that defines Section and Quarter IDs, display labels, and per-Section view/update roles. Its normalized data includes a `sectionPolicies` map keyed by immutable `sectionId`, so Firestore rules can authoritatively read policy without iterating an array. Both dashboard entries render from this configuration. Callable Functions reload it before every update or structural request and remain the only persistence boundary.

**Tech Stack:** Static HTML/ESM browser app, Firebase Firestore, Firebase Functions v2, CommonJS Node core, Node test runner.

## Global Constraints

- Do not push or deploy; this remains local preview work.
- `admin` and `executive` apply structural changes directly; another permitted role submits an approval request.
- Every structural action has a mandatory reason and append-only audit record.
- Section and Quarter IDs are immutable. Only their labels and access policies are configurable.
- Authorization never uses a displayed label or client-supplied permissions.
- The embedded `weeks` document does not provide database-level Section confidentiality.
